Astro Stacker 3.1 - User Manual

Astro Stacker is an application for stacking astronomical images from DSLR and mirrorless cameras, astronomy cameras, and smart telescopes. It loads a sequence of Light frames, optionally applies Flat/Bias/Dark calibration, aligns the frames on stars or a comet, stacks the result, and provides preview tools for inspection and export.

The program starts with a short splash screen and opens maximized so the preview, side panels, and diagnostic table have enough room. The linear XISF/FIT output remains suitable for further processing. Most controls in the right panel affect the preview and PNG/JPG/TIFF export, not the original linear data.

1. What Astro Stacker Does

  • Stacks Light frames from a selected folder.
  • Supports PixInsight XISF, FIT/FITS, common camera RAW files, and standard image formats.
  • Applies optional Flat, Bias, and Dark calibration.
  • Aligns frames by translation, affine ECC, stars with RANSAC, perspective star matching, or comet motion.
  • Allows automatic or manual reference frame selection.
  • Filters the best frames by quality score.
  • Shows a diagnostic frame quality table.
  • Detects satellite trails and masks only the affected pixels during stacking.
  • Saves and loads settings profiles.
  • Provides preview tools with histogram, curves, color controls, background balancing, and Balance.
  • Exports linear XISF/FIT or visually adjusted PNG/JPG/TIFF.
  • Controls one or more ASCOM Alpaca cameras and sends their exposures to a shared Live Stack folder.
  • Includes the PixInsight wrapper package for users who want to run the CLI stacker from PixInsight.

2. Quick Start for Normal Stacking

  1. Click Choose folder and select a folder with Light frames.
  2. Enable RAW only if the folder also contains JPG/PNG/BMP/TIFF preview files.
  3. For a normal deep-sky stack, keep these settings:
  • Alignment: Star alignment - stars + RANSAC
  • Stacking: Median
  • Automatically choose best reference: enabled
  • Use only best frames: enabled as needed
  1. Click Start stacking.
  2. After stacking, check the preview, diagnostic table, and the used/excluded frame summary.
  3. Save the result with File > Save result as.

2A. ASCOM Alpaca Cameras and Multi-Camera Live Stack

Open Camera > ASCOM Alpaca cameras.... The same window is used for one camera or for a synchronized group of cameras.

  1. Enter the Alpaca server address and port. The default is 10.42.42.1:80.
  2. Click Refresh to discover available cameras.
  3. Enable Use for every camera that should take part in the sequence.
  4. Click Connect selected.
  5. Set exposure, count, interval, gain, binning, cooling, and target temperature.
  6. Select one shared output folder. Enable automatic Live Stack startup when required.
  7. Click Start synchronized sequence.

The Camera/ID file column combines the camera name with its unique file identifier. The identifier is included in generated file names and in the FIT CAMID keyword, so frames from several identical cameras remain distinguishable.

Each selected camera runs independently, reports its own status, and writes completed exposures into the same folder. Live Stack waits for complete files and can safely process frames arriving from several cameras at nearly the same time.

All cameras must be reachable from the computer at the same time, normally through one LAN, Wi-Fi router, or multiple correctly configured network adapters. Direct camera Wi-Fi may disconnect normal Internet access. Camera discovery uses Alpaca discovery plus a direct query to the entered address.

Astro Stacker accepts Alpaca JSON images and binary ImageBytes responses. Bayer FIT metadata supplied by the camera is preserved for automatic debayering. The interface has been tested with Pegasus SmartEye and Seestar cameras.

3. Left Panel - Stacking Controls

The left and right side panels can be collapsed with the small arrow button in

the top-left corner of each panel. When a panel is hidden, only a narrow arrow

rail remains. This gives more space to the central preview and the Frame quality

table on smaller displays.

Help and Updates

Astro Stacker checks GitHub Releases for a newer version shortly after startup. If a newer release is available, it shows a notification and can open the download page. You can also run the check manually from Help > Check for updates.

Folder and Preview Selection

Choose folder loads the working folder. The file list shows Light, Flat, Bias, and Dark frames. After stacking, the list marks:

  • * the reference frame,
  • x excluded frames.

The file list can also be used to inspect individual frames.

RAW Only

When enabled, RAW only keeps XISF, FIT/FITS and camera RAW files such as ARW, CR2, CR3, and similar formats. It excludes JPG, PNG, BMP, and TIFF files. This is useful when a camera or smart telescope stores both working frames and preview images in the same folder.

XISF files from PixInsight can be used as Light or calibration frames. Compressed XISF files are supported. XISF without explicit CFA/Bayer metadata is treated as monochrome; a CFA XISF is debayered automatically or according to the manually selected Bayer pattern.

For Bayer XISF/FIT/FITS sequences, calibration is applied to the sensor mosaic before debayering. Large FIT headers are inspected without loading the full image, and the actual loading, debayering, and alignment run in parallel worker processes when multiprocessing is enabled.

Alignment Modes

Translation

Fast shift-only alignment. Suitable for small drift without rotation.

Calibration/no alignment

Stacks calibration frames pixel-to-pixel. Bayer FIT/XISF/RAW frames stay as the original 2D sensor mosaic and are not debayered. This is the correct mode for creating Master Bias, Master Dark, or Master Flat FIT files. The preview may display the 2D master as neutral monochrome RGB only for screen display; the saved FIT remains mono/mosaic.

ECC affine

Uses image correlation to estimate shift, rotation, and scale. It can be slower and may not work well on weak objects.

Star alignment + RANSAC

The recommended main mode. The program detects stars, matches them against the reference frame, and estimates the transformation with RANSAC. Frames without a valid alignment are rejected.

Star perspective - stars + corners

An experimental mode for more complex distortion. Use it only when normal RANSAC alignment is not enough.

Comet alignment

Stacks on the comet position.

Star + Comet

Creates separate star and comet stack outputs.

Automatic and Manual Reference

Automatically choose best reference selects the frame with the best quality score. The score is based mainly on sharpness and detected star count.

If automatic reference selection does not work well for a specific sequence, select a suitable frame in the file list and click Use current frame as reference. This disables automatic reference selection and uses the selected frame instead.

Quality Filter

The program computes a quality score for each Light frame. The score combines:

  • frame sharpness,
  • detected star count.

Keep defines what percentage of the best frames will be used. For example, 80% keeps the best 80% and excludes the worst 20% before alignment.

Important: a frame can pass the quality filter and still be rejected later if alignment fails.

Max. Star Drift

Defines the maximum expected star displacement relative to the reference frame. Normal sequences can use a smaller value. EAA, dithering, or smart telescope sequences may need a larger value.

Ignore Border

Ignores the image borders during star detection. This helps when the edges contain branches, noise, vignetting, black corners after rotation, or other distracting structures.

Strict Star Filter

Applies stricter star shape filtering. It helps suppress branches, trails, and elongated artifacts. If too many usable frames are rejected, disabling this option can sometimes help.

Satellite Trail

Detects long straight satellite or aircraft trails and marks suspect frames in the Frame quality table. During stacking, Astro Stacker masks the complete detected line from edge to edge. Only the trail pixels are excluded; the unaffected part of the exposure remains available for integration.

Live Stack Frame Rejection

Live Stack uses online sigma rejection to suppress hot pixels, cosmic rays, aircraft, and satellite trails while frames are arriving. The Reject elongated stars option can reject a whole Live Stack frame when the measured stars are too stretched. This helps when wind, tracking errors, or mount vibration create unusable frames.

4. Diagnostic Frame Quality Table

The lower part of the window contains the Frame quality table. It helps explain why frames were used or excluded.

Columns:

  • # frame order,
  • File file name,
  • Score overall quality score,
  • Stars detected star count,
  • Sharpness Laplacian variance sharpness metric,
  • Status frame status,
  • Ref reference marker.

Typical statuses:

  • Used - the frame was stacked.
  • Reference - the reference frame.
  • Excluded by quality - the frame failed the quality filter.
  • Rejected alignment - the frame passed quality filtering, but alignment failed.
  • Skipped - the frame was not part of the stack, for example a calibration or unused file type.

Clicking a row switches the preview to that frame.

Reviewing Frames Before Stacking

Review frames before stacking adds a manual review step before the actual stack begins.

Workflow:

  1. Enable Review frames before stacking.
  2. Click Start stacking.
  3. The program only computes frame quality, chooses the reference, and fills the Frame Quality table.
  4. Select a frame in the table that you do not want to use.
  5. Press Space. The status changes to Excluded and the row turns red.
  6. Press Space again to allow the frame again.
  7. Click Continue stacking.

The reference frame cannot be excluded with Space. If you want to exclude the reference, choose another reference frame first.

Manual exclusion has priority over the automatic selection. The quality filter still makes the initial selection, but before final alignment you can manually remove additional frames.

5. Flat, Bias, and Dark Calibration

Calibration frames can be used in two ways:

  • automatically, if they are in appropriately named folders,
  • manually with the Flat, Bias, and Dark buttons in the right panel.

Calibration is applied before alignment and stacking for Light frames.

If you want to create a master calibration file, choose Calibration frames - no alignment. In this mode Astro Stacker does not apply Flat/Bias/Dark from the right panel, does not run quality filtering or auto-reference selection, and does not debayer Bayer data. The result can be saved as a linear Master FIT.

Reset calibration removes the selected calibration frames.

6. Comet Stacking

For comet stacking, the most reliable method is to mark the comet position in two frames.

  1. Select the folder with the sequence.
  2. Click Comet First.
  3. The program loads the first frame. Click the comet nucleus in the preview.
  4. Click Comet Last.
  5. The program loads the last frame. Click the comet nucleus again.
  6. The program now knows the comet motion between the beginning and end of the sequence.
  7. Select Comet alignment or Star + Comet.
  8. Start stacking.

Comet Clear removes both saved comet positions.

Comet options:

  • Max. comet motion - maximum expected comet movement.
  • Refine comet position - refines the nucleus position with local correlation.
  • Comet template - template size around the nucleus.
  • Comet search - how far from the predicted position the nucleus may be searched.

7. Settings Profiles

Settings profiles are saved as JSON files.

Use:

  • File > Save settings profile
  • File > Load settings profile

After selecting a working folder with Choose folder, the result and settings-profile save dialogs open directly in that folder.

A profile stores stacking, calibration, comet, and preview settings. Profiles are useful for different workflows, such as:

  • DSLR RAW,
  • Seestar/Vespera EAA,
  • comet processing,
  • calibration frame stacking,
  • linear FIT workflows.

8. Right Panel - Preview and Adjustments

The right panel is used for inspection and visual adjustments. The controls are grouped into tabs for color, correction, and AI tools. These controls mainly affect preview and PNG/JPG/TIFF export. Linear XISF/FIT data remains suitable for further processing.

After completion, the stack is displayed linearly and may look almost black. Balance stretches the preview only and does not modify the linear data.

Use the small arrow in the top-left corner of the panel to hide the whole right

panel temporarily. Click the remaining arrow rail to show it again.

Show Stacked Image

Restores the preview to the original stacked image without crop, neutralization, or visual adjustments.

Balance

Balance automatically sets a reasonable preview for a linear image:

  • neutralizes the background for preview,
  • sets the black point,
  • sets gamma,
  • makes weak linear data easier to inspect.

Balance does not modify the linear FIT data. It is a preview helper.

Black Point, White Point, Gamma

Basic display curve controls:

  • Black point moves the black level.
  • White point sets the bright level.
  • Gamma adjusts midtones.

Black point and White point can be adjusted in fine 0.05 increments, for example to 2.25.

STF Strength

Controls the Screen Transfer Function intensity for the preview and visual PNG/JPG/TIFF export only. The displayed scale runs normally from left to right: 0 on the left and 10 on the right. Its effect is intentionally reversed: 0 gives the strongest STF for faint structures, 5 is the standard preview, and 10 gives the gentlest STF. Linear XISF/FIT export is unchanged.

Classic and AI DRUNet Denoising

Classic uses the built-in CPU denoiser. AI DRUNet uses an ONNX neural-network model and normally produces stronger luminance and color-noise reduction.

  • models/drunet_color.onnx is the default model for RGB images.
  • models/drunet_gray.onnx is available for monochrome images.
  • After the first calculation, Astro Stacker stores an adaptive noise map. Gamma, STF, contrast, color, Balance, gradient removal, and other preview controls reuse this map instead of running the neural network again.
  • Changing the denoise strength, model, source image, or crop starts a new denoise calculation.

ONNX Runtime prefers CoreML on macOS and CUDA or DirectML on Windows when available. Otherwise it automatically uses the CPU. CPU processing is slower but remains fully functional.

AI Star Deconvolution

AI Star Deconvolution sharpens and gently reduces compact stars with the bundled Cosmic Clarity stellar ONNX model. The Star size control changes the radius of the star-only protection mask so broad galaxy and nebula structures are not processed as stars.

  • models/cosmic_clarity_stellar.onnx is selected automatically.
  • The first calculation is cached. Strength, Star size, STF, gamma, contrast, and color changes reuse the cached model result.
  • CoreML is preferred on macOS; CUDA or DirectML is preferred on Windows when available.
  • The effect is applied only to the preview and visual PNG/JPG/TIFF export. Linear XISF/FIT output remains unchanged.

Vignette Removal

Gently brightens edges and corners. It is not a replacement for real Flat calibration, but it can help with quick visual inspection.

Synthetic Flat

Estimates a smooth background from the stacked image and uses it as a gentle synthetic flat. Use carefully, especially with large nebulae, where the program may mistake real nebulosity for background.

Color Background Correction

Suppresses smooth color casts in the background per RGB channel. This is useful for strongly unbalanced images, such as pink or purple backgrounds from smart telescopes.

SCNR Green

Gently suppresses only excess green using a Maximum Neutral method. Green moves toward the stronger red or blue channel, preserving more real green-cyan nebulosity than reducing green to the R/B average.

Contrast, Saturation, RGB

  • Contrast adjusts preview contrast.
  • Saturation adjusts color intensity.
  • Red, Green, Blue manually adjust individual channels.

Histogram

The histogram shows luminance and RGB channels. It helps verify whether data is clipped in shadows or highlights.

Turn Angle

Rotates the current image by the selected angle. This is useful when a stacked image is slightly tilted and you want to straighten the field before choosing the final crop.

The Select button enables manual crop selection. Drag a rectangle directly

in the preview; the selected area is kept and everything outside it is removed.

This is useful when only one side or an irregular border area needs to be

removed. Manual crop affects the current preview/opened image or stacked image,

after any Turn Angle rotation if one was applied.

Undo

The Undo button restores the previous preview-processing state. It is useful after actions such as Balance, Auto WB, denoise, Turn Angle, or manual crop.

Auto White Balance

Attempts automatic white balance on neutral/gray regions. For some astro images, manual RGB adjustment or Color background correction may work better.

Neutralize Background

Attempts to equalize the background color. It selects background by geometry, luminance, and absence of stars rather than requiring similarly strong RGB channels. This improves images with a very weak blue or another color channel. Additive channel offsets are used so noise in the weak channel is not unnecessarily amplified. It works best when enough real background is visible outside large nebulosity.

Flip Horizontal / Vertical

Flips the preview horizontally or vertically.

Fit and 1:1

  • Fit fits the image to the window.
  • 1:1 shows real pixels.

9. PixInsight Wrapper

The release package includes an optional AS_Stacker PixInsight wrapper. It can run the CLI stacker from PixInsight and open the resulting FIT file after completion.

The wrapper needs a Python environment with the required packages installed.

The wrapper package in version 3.1 includes the current stacking engine, the Satellite trail option, both DRUNet models, and the AI Star Deconvolution stellar model. Keep the complete AS_Stacker_PI_Plugin folder together. The CLI wrapper produces linear FIT output, so preview-only denoise and star deconvolution are not applied to the stacked FIT file.

10. Recommended Workflows

Normal Deep-Sky Sequence

  • Star alignment + RANSAC
  • Median
  • Auto reference enabled
  • Quality filter 80-100%
  • Strict star filter enabled

Large Drift or Dithering

  • Increase Max. star drift.
  • Try a manual reference from the middle of the sequence.
  • Use the diagnostic table to see whether frames fail quality filtering or alignment.

Smart Telescopes and EAA

  • Enable RAW only if preview images are mixed with stackable frames.
  • Use Turn Angle and Select crop for rotation and dark corners.
  • Use Color background correction for pink or purple backgrounds.

Comets

  • Mark both first and last comet positions.
  • Use Comet alignment or Star + Comet.
  • Keep comet refinement enabled for weak nuclei.

11. Troubleshooting

Too Many Frames Are Rejected

Check the Frame quality table:

  • if frames are Excluded by quality, reduce quality filtering or increase Keep %;
  • if frames are Rejected alignment, increase Max. star drift, change the reference frame, or try another alignment mode.

The Image Has Strange Colors

Try, in order:

  1. Balance,
  2. Color background correction,
  3. Neutralize background,
  4. manual RGB adjustments,
  5. crop the edges and neutralize again.

The Result Has Dark Corners

This is common with rotation, dithering, or large drift. Use Turn Angle, Select crop, or crop the result later in another editor.

FIT Output Looks Dark

This is normal. FIT output is linear and needs stretching. Use Balance for preview, or process the output in PixInsight, Siril, or another astrophotography editor.

12. Installation

Windows Installer

ASCOM Alpaca camera support uses standard network components and requires no additional Python package.

  1. Run AstroStacker31_Setup.exe.
  2. The installer contains the CPU application and, optionally, the NVIDIA CUDA build. Choose CUDA support only on a Windows computer with a supported NVIDIA GPU and a current NVIDIA driver.
  3. The installer also bundles the Microsoft Visual C++ Runtime and runs it silently when needed.
  4. A smaller AstroStacker31_Lite_Setup.exe can be built for users who want a CPU-only package without AI denoise, CUDA, ONNX Runtime, PyTorch, or the PixInsight wrapper.

macOS DMG

  1. Open AstroStacker31_macOS.dmg.
  2. Drag AstroStacker.app to Applications.
  3. The app bundle contains ONNX Runtime for AI tools and PyTorch for Apple Metal/MPS stacking acceleration.

Source Installation for Testing

Developers can still use install_astro_stacker_macos.command on macOS, or create a Python environment manually on Windows. End users should normally use the standalone installer or DMG.

The standalone Windows installer and macOS DMG already contain the splash image, English/Czech manuals, both DRUNet models, the AI Star Deconvolution stellar model, and the PixInsight wrapper package. They do not require a separate Python installation.
